# CS-210 Journal Repo

  | Summarize the project and what problem it was solving.
This project was a simple calculator that would calculate the amount of interest that would accumulate for the balance of a savings account over the course of _x_ amount of years, based on the initial balance, interest rate, and how much money is deposited each passing month.

  | What did you do particularly well?
I think the presentation of the table it prints out once it calculates everything is pretty snazzy--makes it all particularly easy to read and follow.

  | Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
I think an upper level of a UI could be implemented with perhaps add a few more calculator "engines" to the project. Then have a "multitool" of various financial calculators.

  | Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
I found getting the fancy table formatting implemented correctly was pretty challenging. Two super obvious resources for support would be YouTube and the big AI products, such as ChatGPT, Copilot, or Gemini. Those services, alone, have taught me so much. I cannot imagine coding prior to their availability, haha! They're incredibly convenient and great at getting me out of a pinch.

What skills from this project will be particularly transferable to other projects or course work?
This goes for most of the projects in CS-210, but I think the biggest lesson learned is how modularity in design helps tremendously when it comes to flexibility in expanding and scaling larger. I recognize now just how many small, simple considerations come up during development that need to be answered sooner than later to prevent backtracking and rewriting later

How did you make this program maintainable, readable, and adaptable?
I just try to find super "common sense" approaches to my code's design and solutions to problems. Even if there is a way that is technically better for one reason or another, if my design is not necessarily hurting anything and is also more readable and intuitive, I think it is worth the cost of some sort of, say, some efficiency.
